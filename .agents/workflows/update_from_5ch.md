# 5ch・YouTube等の外部情報からWiki反映ワークフロー

ユーザーから5ch（または類似の掲示板）の書き込みログ、またはYouTube動画のURLリストが提供された場合、以下の手順に沿ってWiki（`docs/` 配下）を更新してください。

---

## 0. ソース種別の判定

提供されたソースの種別に応じて情報抽出手段を切り替えます。

| ソース種別 | 情報抽出方法 |
|-----------|-------------|
| **5chログ（テキスト）** | ユーザーが貼り付けたテキストを直接精読する |
| **YouTube動画URL** | `yt-dlp` 等でタイトルと概要欄を取得し、そこから情報を抽出する。動画本編の内容を推測で補わない |
| **公式サイトURL** | `read_url_content` 等でページの内容を取得する |

## 1. 情報の抽出と分類
提供されたテキストログ・動画概要欄を精読し、攻略や仕様に関する情報を以下のカテゴリに分類・抽出します。
- クラス（役目）、武器、スキルに関する情報
- モード（奇譚、九死、攻城、大禍等）に関する情報
- テクニック、立ち回り、システム・設定上のTIPS
- バグや不具合の報告
- **ビルド・構成例**（特定プレイヤーの構築パターンとして扱い、「最強」「おすすめ」とは断言しないこと）

## 2. ゴーストオブツシマ（前作）との混同チェック
前作『Ghost of Tsushima: Legends』の仕様と今作『Ghost of Yotei: Legends』の仕様が混同されていないかチェックしてください。前作の常識が今作では通じない（例：牢人の全体蘇生の有無など）点を明確に分けます。

## 3. 隠語・短縮語の正式名称化
5ch特有の隠語や短縮語は、Wikiの記載ガイドラインに沿って日本の正式ゲーム用語に翻訳・統一します。
- レジェ / レジェンズ / オン → 冥人奇譚
- サバイバル / サバ → 九死
- ストーリー / 奇譚 → 奇譚（ストーリーミッション）
- お金 / ポイント → 勾玉（または対応するゲーム内通貨）
- 外人 / 野良 → マッチメイキング / 野良プレイヤー

YouTube動画では英語表記が多いため、以下も注意します。
- True Aim Yumi → 必中の弓
- Spirit Kunai → 神威の苦無
- Dual Katanas → 二刀
- Hunter → 弓取
- Samurai → 侍
- Mercenary / Ronin → 用心棒
- Shinobi / Assassin → 忍
- Nightmare → 百鬼
- Platinum → 白金
- Survival → 九死

## 4. 既存情報群との差分チェックと修正
抽出した情報をもとに、既存の `docs/` 配下の Markdown ファイル群（`classes/` `modes/` `gear/` `guide/` 等）を調査（`grep_search` や `view_file`）します。
- **誤情報の修正**: これまでの検証結果（推測）が新たな報告によって覆された場合、過去の記述（間違った仕様解説など）を速やかに修正します。
- **新情報の追記**: 新たに判明したギミックや有効な立ち回り、武器の仕様などを該当ページに追記します。
- **勝手な攻略情報を作らない**: あくまで「報告された事実・実測データ」を網羅することが目的です。AI自身が勝手に「これが最適解だろう」と論評を作ってはいけません。
- **YouTubeの情報はあくまで「参考例」**: ビルドや稼ぎルートは `!!! info "参考例：〇〇"` のブロックで紹介し、特定のプレイスタイルを「最強」「おすすめ」と断言しないこと。

## 5. 出典（ソース）の管理

### 5chの場合
- `docs/sources/5ch-threads.md` にスレッドURLを追記する
- 追記した各ファイルの `## ソース` 欄に `[5chスレ PartN](../sources/5ch-threads.md)` のリンクを追加する
- 記事本文中には `[（5ch報告）](../sources/5ch-threads.md)` の形式で記載する

### YouTubeの場合
- `docs/sources/youtube.md` にURLとテーマ一覧を追記する
- 追記した各ファイルの `## ソース` 欄に `[YouTube検証・構築例](../sources/youtube.md)` のリンクを追加する
- 記事本文中には `[（YouTube検証例）](../sources/youtube.md)` や `[（YouTube構築例）](../sources/youtube.md)` の形式で記載する

### 全ページに漏れなく追記
出典リンクの追加は**全ファイル一括で行い、漏れがないようにする**こと。`sed` やスクリプトでの一括処理を推奨。

## 6. 計画の作成と承認（Implementation Plan）
全ての抽出と差分チェックが完了したら、`implementation_plan.md` に更新予定のリスト（どのファイルのどの部分をどう直すか）を作成し、ユーザーに提示します。
承認が得られてから、実際のファイル置換（`multi_replace_file_content` 等）を実行してください。

# 5ch情報のWiki反映ワークフロー

ユーザーから5ch（または類似の掲示板）の書き込みログが提供された場合、以下の手順に沿ってWiki（`docs/` 配下）を更新してください。

## 1. 情報の抽出と分類
提供されたテキストログを精読し、攻略や仕様に関する情報を以下のカテゴリに分類・抽出します。
- クラス（役目）、武器、スキルに関する情報
- モード（奇譚、九死、攻城、大禍等）に関する情報
- テクニック、立ち回り、システム・設定上のTIPS
- バグや不具合の報告

## 2. ゴーストオブツシマ（前作）との混同チェック
前作『Ghost of Tsushima: Legends』の仕様と今作『Ghost of Yotei: Legends』の仕様が混同されていないかチェックしてください。前作の常識が今作では通じない（例：牢人の全体蘇生の有無など）点を明確に分けます。

## 3. 隠語・短縮語の正式名称化
5ch特有の隠語や短縮語は、Wikiの記載ガイドラインに沿って日本の正式ゲーム用語に翻訳・統一します。
- レジェ / レジェンズ / オン → 冥人奇譚
- サバイバル / サバ → 九死
- ストーリー / 奇譚 → 奇譚（ストーリーミッション）
- お金 / ポイント → 勾玉（または対応するゲーム内通貨）
- 外人 / 野良 → マッチメイキング / 野良プレイヤー

## 4. 既存情報群との差分チェックと修正
抽出した情報をもとに、既存の `docs/` 配下の Markdown ファイル群（`classes/` `modes/` `gear/` `guide/` 等）を調査（`grep_search` や `view_file`）します。
- **誤情報の修正**: これまでの検証結果（推測）が5chの報告によって覆された場合、過去の記述（間違った仕様解説など）を速やかに修正します。
- **新情報の追記**: 新たに判明したギミックや有効な立ち回り、武器の仕様などを該当ページに追記します。
- **勝手な攻略情報を作らない**: あくまで「報告された事実・実測データ」を網羅することが目的です。AI自身が勝手に「これが最適解だろう」と論評を作ってはいけません。

## 5. 計画の作成と承認（Implementation Plan）
全ての抽出と差分チェックが完了したら、`implementation_plan.md` に更新予定のリスト（どのファイルのどの部分をどう直すか）を作成し、`notify_user` を用いてユーザーに提示します。
承認が得られてから、実際のファイル置換（`multi_replace_file_content` 等）を実行してください。
